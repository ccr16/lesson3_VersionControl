Highlight #1 Version Control

Version control records charges to files or sets of files so that you are able to revisit specific versions later on. With version control, you can see every step you have taken to get to the completed version. This means that you are able to see the beginning stages, who caused a problem, and how much the project has progressed. There are multiple ways to utilize version control. Majority of people copy files. There are several different types of version control. Local version control, centralized version control, and distributed version control systems.

Highlight #2 Git

Git is the version control system that we will be using in class. It is a distributed version control system. This means that every user has a copy of the data stored on their computer. You are able to work offline due to it being a local repository. There are basic git commands that we will be using in this class. The three steps are add, commit, and push. This is important to this class because we will be working on the same files, which gives everyone access to read them. Currently in class, we connected out Cloud 9 accounts with GitHub using SSH remotes.

Highlight #3 More on Version Control

Version control systems combine project files to one major repository. This makes it simpler for those who are working on shared files to make changes. Version control systems automatically make note of when files were changed, who changed them, what was changed, and makes sure that the files have all been changed. Two people may work on the same file at once. It is also possible for those working to view the beginning, middle, and end stages of the files. The version control system also backs up all the files. There are two types of Repositories, remote and local. Remote repositories reside on a remote server that is shared among multiples. A local repository is on a local machine of an individual user. .
